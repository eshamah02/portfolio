SkinMax

Code not available due to liability reasons

Skinmax is an AI-powered skincare recommendation system. By taking in some basic information and a selfie of the user, it's able to provide personalized skincare guidance and product recommendations in seconds. Users are able to track their progress and see exactly how much their skin has improved from the last time they submitted a photo.

I'm fortunate enough to be currently working with SkinMax and helping them with product design and implementation. Here are some things I changed about the original design and my thought process behind it.

1. Home Page/Landing Page

The original home page wasn't poorly designed, but it wasn't as appealing to users as it could be.

- despite mock ups being available that would explain what the app was and how it worked, users had to scroll to see it even in desktop view

- 'how it works' section was unnecessary and seemed childish with emojis

- no defining features or characteristics that provided brand identity

- on desktop, users may have to scroll down to see the 'Try it Free' call-to-action, demotivator

Instead of rolling with the informational vibe that the original home page had, I wanted to make it more of a landing page to get users to the next step quickly. These are the changes I made to make that happen.

- condensed text areas so user can see the mock ups in desktop view, got rid of the 'how it works' section

- call to action to log in or try the application for free is highlighted and eye catching

- had and that had previously been chosen to provide brand identity, so used that font and pushed orange throughout

2. Flow

The original flow worked but wasn't easy to progress through for the user group that isn't familiar with skincare. Because of the large, users had to scroll down on mobile for every question if they wanted to get to the text box and submit, slowing down the flow. Here is what I did to mediate this.

- got rid of the big orange 'S' to make room, this made it a lot easier to step through the flow (especially on mobile)

- made the 'type of skin' a set of options to choose from instead of a text box for users who don't know the different types of skin

As an aside, I also added an for users to input what they were specifically looking to improve so our AI model could give suggestions geared towards those.

For my next steps, I'm going to change the skincare routine and improvement questions to also have a set of options to choose from rather than having a text input to speed up the flow.

3. Responsive Web Design

Designing with a mobile-first design is always a really good idea! But the original pages of SkinMax seemed to be designed with a mobile-only design. Here's what I mean.

- these screenshots were taken on a desktop, however the page only fills what would be visible on a mobile

- there's a navigation bar on all screens that is more associated with mobile apps rather than web applications, which ends up looking kind of odd when you navigate to this website on desktop or mobile

Here's what I did to mediate these issues.

- made all the pages responsive for all screens (also did a bunch of redesigns for all previously existing pages to make them more user friendly! see if you can spot them!)

- added a navigation bar at the top for desktop view

- added a bar for mobile view

4. Profile Page

There was initially no profile page on this application. This meant that users didn't have a dedicated place to update their information, log out of their account, and even delete their account.

So I made one!

For my next steps, I plan to add a profile photo and a spot so users can see how many credits they have.
